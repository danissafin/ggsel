# GGSEL Seller Bot + Telegram Mini App v4.8

Бот дублирует сообщения покупателей в Telegram и открывает защищённую графическую панель управления GGSEL прямо внутри Telegram.

## Что есть в панели

- обзор баланса, продаж и состояния каталога;
- товары с поиском и фильтрами: все, активные, приостановленные, без остатка, архив;
- карточка товара и просмотр содержимого склада;
- добавление содержимого вручную или из TXT-файла;
- изменение цены, названия, категории, описания и режима автовыдачи;
- активация, пауза и архивирование товара;
- массовая активация, пауза и архивирование выбранных товаров;
- создание нового оффера;
- поиск заказа по номеру и последние продажи;
- баланс, чеки и сумма поступлений за период;
- отзывы, категории, чаты и ответы покупателям;
- история всех изменений из Mini App.

Все изменения требуют подтверждения. API-панель принимает запросы только с корректной подписью Telegram Mini App и только от `OWNER_ID`.

## Обновление существующего проекта

1. Распакуйте архив.
2. Замените содержимое GitHub-репозитория файлами из папки `ggsel_bot_miniapp`.
3. Убедитесь, что папки `templates` и `static` тоже попали в GitHub.
4. Выполните commit и push.
5. Дождитесь успешного deploy в Railway.
6. Откройте настройку webhook повторно:

```text
https://ggsel-ggsel.up.railway.app/setup-telegram-webhook?secret=ВАШ_SETUP_SECRET
```

Этот URL одновременно:

- устанавливает webhook Telegram;
- создаёт постоянную кнопку меню **«Управление»**;
- добавляет основные команды бота.

## URL переадресации сообщений GGSEL

```text
https://ggsel-ggsel.up.railway.app/ggsel?secret=ВАШ_GGSEL_WEBHOOK_SECRET
```

Метод — `POST`.

## Запуск панели

Откройте бота и нажмите кнопку меню **«Управление»** либо отправьте `/start` и нажмите **«Открыть панель управления»**.

Обычное открытие `/app` в браузере не даёт доступ к API: сервер требует подписанные данные запуска Telegram.

## Работа с приостановленным товаром

1. Откройте вкладку **Товары**.
2. Выберите фильтр **Приостановлены**.
3. Откройте карточку товара.
4. Нажмите **Добавить содержимое** и вставьте ключи или выберите TXT-файл.
5. После загрузки нажмите **Включить**.

## Важные замечания

- Массовые операции GGSEL V2 могут возвращать `job_id`: выполнение происходит асинхронно на стороне GGSEL.
- Отображение статуса обновляется после нажатия кнопки обновления; каталог кэшируется на 30 секунд.
- За один запрос Mini App принимает до 2000 строк и отправляет их в GGSEL пакетами по 100.
- API-ключ не передаётся в браузер: он остаётся только в Railway Variables.
- Токен Telegram, опубликованный где-либо ранее, необходимо перевыпустить через BotFather.

## Если вкладка «Товары» показывает GGSEL V2 HTTP 500

В версии 4.1 загрузка офферов использует небольшой размер страницы и автоматически
пробует совместимые варианты пагинации (`limit`, `per_page`, только `page`, а для
первой страницы — запрос без параметров). Это обходит ошибку GGSEL, при которой
слишком большой или неподдерживаемый лимит иногда возвращает HTTP 500.

Необязательная переменная Railway:

```text
MINIAPP_OFFERS_API_PAGE_SIZE=20
```

Рекомендуемый диапазон — 10–30. Значения выше 50 приложение всё равно ограничивает.
После обновления файлов сделайте Redeploy и в панели нажмите «Обновить».


## Исправление v4.4

Каталог товаров загружается по всем страницам V1 с использованием полей `pages` и `cnt_goods`. Исправлена ситуация, когда GGSEL возвращал по 20 строк, несмотря на `count=100`, из-за чего панель показывала только первую страницу.


## Исправление v4.4

Карточка «Подробнее» теперь объединяет данные из полного V1-каталога, V1-карточки товара и V2-карточки. Если V2 возвращает только ID, название, цена, остаток и статус больше не обнуляются.

## v4.5 — исправления заказов, отзывов и чатов

- В карточке заказа UUID больше не используется как название товара. Если GGSEL возвращает UUID в поле `name`, бот подтягивает название по `item_id` из каталога товара или локальной истории продаж.
- Статусы заказа расшифровываются: 1 — создан, 2 — отменён, 3 — оплачен, 4 — выполнен, 5 — возвращён.
- Прибыль выводится как денежная сумма со знаком валюты.
- Текст отзывов читается из документированного поля `info`; показывается тип отзыва и ответ продавца.
- Список чатов объединяет непрочитанные диалоги из GGSEL API с локальной историей входящих сообщений, сохранённой webhook-ом.
- При открытии диалога сообщения GGSEL объединяются с локально сохранёнными сообщениями; если API чатов временно недоступен, локальная история всё равно отображается.
- Отправленные из Mini App ответы сохраняются в локальную историю.

Важно: официальный метод `GET /api_sellers/api/debates/v2/chats` возвращает список непрочитанных диалогов, а не полный архив. Поэтому старые прочитанные диалоги, которые не проходили через webhook этого бота, API может не вернуть. Новые входящие сообщения сохраняются в Railway Volume и будут доступны в панели постоянно.


## v4.7 — новый дашборд

Вкладка **Обзор** переработана по мотивам кабинета GGSEL и адаптирована под Telegram Mini App:

- периоды: сегодня, вчера, 7 дней, 30 дней, 6 месяцев и произвольные даты;
- фильтр аналитики по конкретному товару;
- сумма продаж, число заказов, сумма к зачислению и средний чек;
- сравнение с предыдущим аналогичным периодом;
- вертикальные графики продаж, зачислений и количества заказов;
- баланс, ожидающие зачисления, активные и приостановленные товары;
- товары без остатка и с низким остатком;
- статистика активных диалогов поддержки;
- топ товаров и быстрый переход к пополнению склада;
- кэш аналитики, чтобы не нагружать GGSEL при каждом открытии панели.

Seller API не отдаёт просмотры карточек, переходы к оплате и конверсию, поэтому эти показатели не рассчитываются и не подменяются выдуманными значениями. Дашборд показывает только данные, которые можно получить из баланса, чеков, продаж, каталога и локальной CRM.

Для длинных периодов рекомендуется:

```env
RECEIPTS_MAX_PAGES=30
DASHBOARD_ANALYTICS_CACHE_SECONDS=60
```

Если за 6–12 месяцев было очень много операций и панель пишет, что данные частичные, увеличьте `RECEIPTS_MAX_PAGES` до 40–60.


## v4.8 — рабочее место продавца

Версия 4.8 добавляет инструменты, которые ускоряют ежедневную работу и не требуют новых разрешений GGSEL API:

- **общий поиск** по товарам, локально сохранённым заказам и чатам;
- кнопка поиска в шапке и быстрый вызов клавишей `/` на компьютере;
- **центр внимания** с товарами на исходе, чатами с метками «Новый»/«Замена» и неуспешными фоновыми операциями;
- счётчик проблем в шапке Mini App;
- блоки «Требует внимания» и «Недавно открывали» на дашборде;
- быстрые действия: найти, пополнить остатки, открыть заказ, перейти в поддержку;
- **избранные товары** и отдельный фильтр `★ Избранные`;
- приватные заметки и метки к заказам: «Проверить», «Замена», «Ждём клиента», `VIP`, «Решено»;
- компактный режим карточек;
- режим скрытия денежных сумм при демонстрации экрана;
- последние открытые товары, заказы и чаты сохраняются в Railway Volume.

Новые таблицы SQLite создаются автоматически при первом запуске. Миграции вручную выполнять не нужно.

### Как пользоваться общим поиском

Нажмите кнопку `⌕` в шапке или карточку **Общий поиск** в разделе «Ещё». Поиск понимает:

- название и ID товара;
- номер заказа, email покупателя и название товара в заказе;
- email, номер заказа, название товара и текст последнего сообщения в чате.

### Избранные

Нажмите звезду на карточке товара или в окне «Подробнее». Избранные позиции доступны через фильтр `★ Избранные` и сохраняются между деплоями.

### Заметки к заказам

Откройте заказ по номеру. Внизу карточки появится блок **Моя заметка**. Заметки хранятся только в вашей базе и не отправляются покупателю или GGSEL.
